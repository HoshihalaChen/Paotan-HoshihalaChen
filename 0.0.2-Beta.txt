【v0.0.2 新增功能（相比 v0.0.1）】

一、模组系统增强
  - 8 个本地模组完整中文数据（名称/NPC/地点/任务/物品/世界观）
  - 模组隔离（PIN）：AI 锁定在选定模组内，禁止串用其他模组内容
  - 模组 → 角色 → 天数 三级存档系统，支持读档回退
  - 每角色最多 10 个存档（FIFO 覆盖），每模组最多 9 个角色

二、属性系统重构
  - 三大独立属性体系：D&D 六维 / 诡秘六维 / COC 八维
  - 初始总属性固定 20 点，由职业+种族固定修饰值决定
  - 设置页滑块自定义每个职业/种族的属性加减值（-3 到 +3）
  - 自定义覆盖持久化到 localStorage，不受游戏重置影响
  - 创建角色时属性只读展示，不可手动修改
  - 雷达图：悬停职业显示职业加成，悬停种族显示种族加成，无悬停显示组合

三、统一检定系统
  - AI 主动询问是否检定（.yes / .no）
  - .yes：代码自动计算：属性调整值 + 熟练加值 + 职业特长 + 装备加成 vs DC
  - .no：固定裸 d20 = 7~9（低于平均水平）
  - 7 级难度预设（极简单 DC5 ~ 传奇 DC25）
  - 22 个职业特长映射 + 装备匹配加成

四、游戏系统
  - 天数周期：随机起始日期 + 每天 4 事件上限 + 自动推进 + 睡眠触发
  - 自动存档：天数+1 时自动触发，半透明提示 3 秒
  - 自动备份：IndexedDB 存储纯文本存档，不受重置影响
  - 离开游戏页 / 切换页面自动存档提示
  - 战斗面板：角色/敌人 HP 实时可视化 + 快速伤害按钮

五、AI 增强
  - 每条回复强制以问句结尾 + 提供 2-3 个编号选项
  - 选项按钮自动解析 AI 回复并渲染可点击按钮
  - 检定请求标准化格式（类型 + DC + 后果）
  - 模组上下文完整注入 System Prompt

六、数据管理
  - IndexedDB 版本标记，不匹配时自动清理重建
  - 3 次重试 + Dexie.delete() 自动恢复
  - 全项目 import 审计 0 缺失
  - JavaScript const 暂存死区 (TDZ) 修复

七、UI/UX
  - 存档管理独立页面（模组→角色→天数三级树）
  - 角色创建雷达图（Chart.js radar 类型）
  - 设置页属性自定义滑块面板
  - 选项按钮解析 AI 编号选项
  - 聊天框滚动到底按钮

【v0.0.1 已有功能（保留）】

一、核心架构
  - Vue 3 Composition API（<script setup>）
  - Vite 5 构建 + HMR
  - Tailwind CSS 复古羊皮纸风格
  - Pinia 跨页面状态管理（10 个 Store）
  - Dexie.js IndexedDB（9 张数据表，v3 schema）
  - Chart.js 数据可视化

二、AI DM 系统
  - DeepSeek V4 PRO API 流式 SSE 打字机回复
  - 记忆机制：角色卡 + 世界观 + 滑动窗口防幻觉
  - 主动引导型 DM：强制问句 + 编号选项

三、7+1 个功能页面
  1. 主页：8 模组卡片 + 系统筛选 + 初始化弹窗 + 游戏锁定
  2. 游戏页：聊天窗口 + d20 骰子 + 选项按钮 + 战斗面板 + 数据统计
  3. 角色页：头像上传 + 六维进度条 + 装备 + NPC 好感
  4. 地图页：地图接口预留 + 只读日期 + 图层勾选
  5. 世界观页：全分类展示 + 模组背景概览 + 力量/势力突出
  6. 日志页：任务列表 + 记事本 + 关联人物
  7. 设置页：API Key + 数据管理 + 属性自定义
  8. 存档页：三级存档树 + 备份管理 + 一键/选择性备份

四、8 个本地模组
  D&D 5E × 6：失落矿坑 / 施特拉德 / 龙枪 / 湮灭之墓 / 冰风谷 / 深水城

  克苏鲁 × 1：COC 7th + SAN 值 + 1920s 新英格兰

五、骰子检定
  - D&D d20 统一检定 / 诡秘序列检定 / COC d100 检定
  - 优势/劣势 + 自定义表达式 + 优劣势
